# Mining Control Center

تطبيق ويندوز للتحكم في أسطول أجهزة **Antminer ASIC** الموزّعة على عدة مواقع، عبر شبكة **Tailscale**.
مراقبة حيّة (هاش/حرارة/مراوح/أونلاين) + أوامر فردية وجماعية (تشغيل/إيقاف/إعادة تشغيل/Reboot) + تنبيهات.

## المتطلبات
- Node.js 20+ و npm
- ويندوز (التطبيق يُغلّف كمثبّت ويندوز)
- وصول للأجهزة عبر الإنترنت — التطبيق يتصل بكل جهاز مباشرة بـ `العنوان:المنفذ` بلا أي برنامج وسيط.
  الطريقة الموصى بها: **فتح منافذ + DDNS** — راجع [دليل الربط المباشر](docs/port-forwarding-setup.md).
  (بديل اختياري لمن يفضّل شبكة مشفّرة: [دليل Tailscale](docs/tailscale-site-setup.md).)

## التشغيل
```bash
npm install
npm start        # يبني التطبيق، يهيّئ المكتبة الأصلية لـ Electron، ثم يفتح البرنامج
```

## الاختبارات
```bash
npm test         # يهيّئ better-sqlite3 لـ Node ثم يشغّل كل اختبارات Vitest
npm run typecheck
```

## البناء (مثبّت ويندوز)
```bash
npm run dist     # ينتج مثبّتاً في مجلد release/
```

## ملاحظة تقنية: المكتبة الأصلية (better-sqlite3)
`better-sqlite3` مكتبة native لها نسخة لـ Node ونسخة لـ Electron:
- `npm test` يعيد بناءها لـ **Node** تلقائياً (عبر `pretest`).
- `npm start` و `npm run dist` يعيدان بناءها لـ **Electron** تلقائياً.
لا حاجة لأي خطوة يدوية ما دمت تستخدم هذه الأوامر.

## البنية
- `src/core/**` — منطق صرف بلا اعتماد على إطار، مغطّى باختبارات (cgminer 4028، محوّلات الفرمور، المراقبة، الأوامر الجماعية، التنبيهات).
- `src/main/**` — عملية Electron الرئيسية (DB، النقل الفعلي TCP/HTTP، الخدمة، IPC).
- `src/preload/**` — جسر IPC آمن.
- `src/renderer/**` — واجهة React (جدول مضغوط مقسّم حسب الموقع).
- `docs/` — وثائق التصميم والخطة ودليل Tailscale.
