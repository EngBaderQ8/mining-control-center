# مركز التحكم بالتعدين — Mining Control Center
### وثيقة التصميم (Design Spec)
- **التاريخ:** 2026-06-29
- **الحالة:** معتمدة من المستخدم (2026-06-29)

---

## 1. الهدف (Goal)

برنامج واحد على جهاز ويندوز الخاص بالمستخدم، يربط كل أجهزة التعدين (Antminer ASIC)
الموزّعة على أكثر من 15 موقعاً، ويعطي:
- **مراقبة حيّة** لكل جهاز (أونلاين/أوفلاين، الهاشريت، الحرارة، المراوح، البول/الوركر، نسبة الأخطاء).
- **أوامر فردية** لكل جهاز (إعادة تشغيل التعدين، إيقاف، تشغيل، Reboot).
- **أوامر جماعية** على مجموعة مختارة أو كل الأجهزة دفعة واحدة.
- **تنبيهات** بسيطة عند انقطاع جهاز أو ارتفاع حرارته أو سقوط هاشه.

يلغي الحاجة للدخول عبر AnyDesk على كل موقع وفتح صفحة كل جهاز يدوياً.

---

## 2. الواقع والقيود (Context & Constraints)

| البند | الواقع |
|------|--------|
| الأجهزة | Antminer **ASIC** (ليست رِجّات كروت GPU). المصطلح المعتمد: «أجهزة ASIC». |
| الفرمور | **خليط:** Stock (Bitmain الأصلي) + معدّل (Braiins OS+ / Vnish / LuxOS). |
| المواقع | أكثر من 15 موقعاً، كل موقع بيت/مكان فيه راوتر منزلي عادي. |
| الشبكة | بدون IP عام ثابت، كل الأجهزة خلف NAT. غالباً عناوين شبكة متطابقة بين المواقع (192.168.x). |
| جهاز دائم بكل موقع | نعم — **لابتوب ويندوز شغّال 24/7** (هو نفسه اللي يُدخل عليه AnyDesk حالياً). |
| جهاز التحكم | كمبيوتر المستخدم — **تطبيق سطح مكتب ويندوز**. |

## 3. المعمارية (Architecture)

```
[ جهاز المستخدم: تطبيق ويندوز (المركز) ]
        │
        │   شبكة Tailscale (مشفّرة، WireGuard) — اتصال خارجي فقط، بدون فتح منافذ
        │
        ├─► موقع 1: لابتوب 24/7 (Tailscale subnet router) ─► Antminer × N (منفذ 4028 + تحكم)
        ├─► موقع 2: لابتوب 24/7 (Tailscale subnet router) ─► Antminer × N
        └─► ... حتى 15+ موقعاً
```

- **القطعة الوحيدة التي نبنيها:** تطبيق الويندوز (المركز). لا وكيل، لا سيرفر سحابي.
- **في كل موقع:** تركيب Tailscale لمرة واحدة على اللابتوب الموجود، وتفعيله كـ
  **subnet router** يعلن شبكة الـ ASIC المحلية.
- **معالجة تطابق الشبكات:** لأن أغلب المواقع تستخدم نفس نطاق العناوين (192.168.x)،
  نستخدم خاصية **Tailscale 4via6** التي تمنح كل موقع مُعرّفاً فريداً وتربط كل جهاز ASIC
  بعنوان IPv6 فريد داخل شبكة Tailscale — بدون تعديل أي راوتر.
- **الاتصال مباشر:** التطبيق يكلّم كل جهاز ASIC مباشرة عبر عنوانه على شبكة Tailscale
  (قراءة عبر منفذ 4028، وتحكّم عبر واجهة الفرمور).

> **اعتماد خارجي واحد:** Tailscale (نسخته المجانية كافية للاستخدام الشخصي). تم اعتماده بدلاً
> من فتح المنافذ (هشّ وغير آمن) أو سيرفر سحابي (تكلفة وتعقيد).

## 4. المكوّنات (Components)

كل مكوّن مسؤوليته واحدة وواضحة وقابل للاختبار باستقلال:

### 4.1 سجل الأجهزة (Device Registry)
- يخزّن: المواقع، وأجهزة كل موقع (الاسم، الموديل، نوع الفرمور، عنوان Tailscale، منفذ التحكم، بيانات الدخول).
- التخزين: قاعدة بيانات محلية **SQLite** على جهاز المستخدم.
- بيانات الدخول (كلمات مرور لوحات الفرمور): **مشفّرة** عبر حماية ويندوز (DPAPI) — لا تُحفظ كنص صريح.

### 4.2 محرّك المراقبة (Monitor Engine)
- يستعلم كل جهاز دورياً (افتراضياً كل **10 ثوانٍ**، قابل للتعديل) عبر **API منفذ 4028**
  (أوامر `summary` / `stats` / `pools` / `devs` المشتركة بين كل أنواع الفرمور).
- يطبّع (normalize) المخرجات إلى نموذج موحّد: الحالة، الهاشريت (لحظي/متوسط)، حرارة اللوحات،
  سرعة المراوح، البول/الوركر، الأخطاء (HW errors)، مدة التشغيل.
- **كشف الأوفلاين:** عبر مهلة/فشل الاستعلامات المتتالية.
- **تحديد التوازي (concurrency limit)** حتى لا يُغرق الاستعلام الشبكة مع ~225 جهازاً.

### 4.3 محوّلات الفرمور (Firmware Drivers — طبقة التحكم)
واجهة موحّدة `DeviceDriver` بالعمليات:
`getStatus()` · `restartMining()` · `stopMining()` · `startMining()` · `reboot()`

| الفرمور | طريقة التحكم |
|--------|--------------|
| Stock (Bitmain) | صفحة الويب CGI (مصادقة digest) لإعادة التشغيل/الـ reboot؛ وإن توفّر أمر API مميّز. |
| Braiins OS+ | واجهة API الخاصة (gRPC الحديثة أو bosminer/SSH) — pause/resume/reboot. |
| Vnish | واجهة HTTP REST (توكن) — reboot/restart/pause/resume. |
| LuxOS | واجهة HTTP/RPC — أوامر التحكم. |

كل فرمور = تطبيق مستقل لنفس الواجهة، فيسهل إضافة فرمور جديد لاحقاً دون لمس بقية النظام.

### 4.4 محرّك الأوامر الجماعية (Bulk Command Engine)
- ينفّذ أمراً على مجموعة مختارة / موقع كامل / كل الأجهزة، **بالتوازي** مع حدّ أعلى.
- **تأكيد إلزامي** قبل الأوامر الخطيرة (إيقاف/Reboot جماعي).
- يعيد نتيجة **نجاح/فشل لكل جهاز** على حدة (لا فشل صامت).

### 4.5 لوحة العرض (UI)
- عرض **مقسّم حسب الموقع**: كل موقع صندوق فيه حالته الإجمالية وأجهزته.
- **جدول مضغوط** (الخيار المعتمد B): كل جهاز ASIC صف فيه — الحالة، الفرمور، الهاش، الحرارة،
  المروحة، الوركر، مدة التشغيل، أزرار الإجراء.
- **شريط علوي ملخّص:** عدد المواقع، أونلاين/أوفلاين، تحذيرات الحرارة، الهاش الكلي.
- **شريط أوامر جماعية** + تحديد متعدد (Checkbox) + «تحديد الكل».
- **بحث وفلترة** (حسب الموقع/الحالة/الفرمور).
- **أوامر على مستوى الموقع** (زر «أوامر الموقع كامل»).

### 4.6 التنبيهات (Alerts) — نسخة أولى بسيطة
- إشعار سطح مكتب عند: جهاز صار أوفلاين، أو تجاوز حرارة حدّاً، أو سقط هاشه تحت حدّ.

---

## 5. تدفّق البيانات (Data Flow)

1. **قراءة:** التطبيق → (Tailscale) → منفذ 4028 لكل جهاز → تطبيع → تحديث الواجهة.
2. **تحكّم:** التطبيق → اختيار المحوّل حسب فرمور الجهاز → (Tailscale) → واجهة التحكم → نتيجة.
3. **تخزين محلي:** سجل الأجهزة + بيانات الدخول المشفّرة + آخر حالة معروفة (لعرض سريع عند الفتح).

---

## 6. الأمان (Security)
- كل المرور عبر Tailscale (تشفير WireGuard)؛ لا تعرّض لأي جهاز على الإنترنت العام.
- بيانات الدخول مشفّرة محلياً عبر DPAPI، غير ظاهرة كنص.
- نوافذ تأكيد للأوامر الجماعية الخطيرة.
- لا فتح منافذ على أي راوتر.

---

## 7. التقنية المعتمدة (Tech Stack)
- **التطبيق:** Electron + TypeScript + واجهة React. *(معتمد من المستخدم)*
- **التخزين:** SQLite محلي.
- **الشبكة:** عميل TCP لمنفذ 4028 + عميل HTTP (digest/REST) لواجهات الفرمور.
- **التغليف:** مُثبّت ويندوز (.exe / NSIS).
- **بدائل مطروحة:** ‎.NET/WPF (ويندوز أصلي) أو Tauri (أخف، أبطأ تطويراً).

## 8. نطاق النسخة الأولى (Scope — v1)

**داخل النسخة الأولى:**
- إعداد الربط عبر Tailscale (توثيق خطوات + اكتشاف الأجهزة).
- سجل الأجهزة والمواقع.
- المراقبة الحيّة (الحالة/الهاش/الحرارة/المروحة/البول/الأخطاء).
- محوّلات الفرمور الأربعة (Stock / Braiins / Vnish / LuxOS) للأوامر الأساسية.
- أوامر فردية + أوامر جماعية مع تأكيد ونتائج لكل جهاز.
- لوحة مقسّمة حسب الموقع بجدول مضغوط + بحث/فلترة.
- تنبيهات بسيطة (أوفلاين/حرارة/سقوط هاش).

**مؤجّل (Out of scope v1):**
- الرسوم البيانية والتحليلات التاريخية.
- حساب الأرباح والربحية.
- الضبط التلقائي (auto-tuning / overclock profiles).
- تحديث الفرمور عن بُعد.
- تطبيق جوال.
- تعدّد المستخدمين والصلاحيات.

## 9. المخاطر والافتراضات (Risks & Assumptions)
- **افتراض:** اللابتوب في كل موقع يبقى شغّالاً 24/7 (هو نقطة الوصول للموقع). انقطاعه = الموقع كامل يصير غير قابل للوصول حتى يعود.
- **افتراض:** الفرمور الستوك يسمح بأوامر التحكم عبر صفحة الويب (بعض إصدارات Bitmain الحديثة تقيّد API/CGI — يُتحقق منه عملياً لكل موديل).
- **خطر:** اختلاف إصدارات الفرمور قد يتطلب تعديلات في المحوّلات؛ نبدأ بموديلات المستخدم الفعلية.
- **خطر:** 4via6 يتطلب دعم IPv6 في عميل Tailscale على جهاز المستخدم (متوفر افتراضياً).
