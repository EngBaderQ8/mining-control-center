# لوحة تحكم المالك (Admin Dashboard)

صفحة ويب على **السيرفر** (VPS) تشرف على كل العملاء: نظرة عامة، تفصيل لكل عميل،
إيقاف/تفعيل/حذف الحسابات، وصحة الوكلاء. **ليست جزءاً من التطبيق** — تعيش في الخادم،
فتشغيلها يحتاج تحديث الخادم (مو التحديث التلقائي للتطبيق).

## الرابط
بعد النشر: `https://144.172.105.191:8443/admin` (المتصفح بيحذّر من الشهادة الذاتية —
اقبلها مؤقتاً لين نركّب شهادة حقيقية). تسجّل الدخول بحساب **المدير** (إيميل مدرَج في
`ADMIN_EMAILS`).

## النشر على الخادم (VPS)
```bash
ssh root@144.172.105.191
cd /path/to/mining-control-center      # مجلد المستودع على الخادم
git pull

# 1) تأكد JWT_SECRET قوي (>=16 حرف). الخادم الآن يرفض الإقلاع بدونه.
#    لو ما تعرف الحالي، ولّد واحد قوي:
#    JWT_SECRET=$(openssl rand -hex 32)

# 2) عرّف إيميل المدير (حسابك) — فقط هذا الإيميل يدخل /admin:
#    ADMIN_EMAILS=baql.almutairi@gmail.com

# 3) أعد التشغيل بالمتغيرات (pm2):
pm2 delete mining 2>/dev/null
PORT=8443 \
JWT_SECRET="<المفتاح القوي>" \
ADMIN_EMAILS="baql.almutairi@gmail.com" \
pm2 start npm --name mining -- run server:start
pm2 save
```

## مهم
- **JWT_SECRET صار إلزامياً وقوياً** — لو كان فاضي أو قصير، الخادم ما يشتغل (هذا حماية:
  المفتاح الضعيف يسمح بتزوير توكن مدير).
- **حساب المدير**: سجّل حساباً عادياً من التطبيق بنفس إيميل `ADMIN_EMAILS`، وبعدها تقدر
  تفتح `/admin`. أي حساب آخر يحاول `/admin/api/*` يرجع له 403.
- **الإيقاف فوري**: إيقاف عميل يمنعه من الدخول **ومن مزامنة البيانات** (يُفصل اتصاله عند
  إعادة الاتصال).
- **سجل التدقيق**: كل إيقاف/حذف يُسجَّل في لوق pm2 (`pm2 logs mining`) مع منفّذ العملية
  ووقتها والـIP.
- **الحذف نهائي** ويمسح كل بيانات العميل (مواقع/أجهزة/حالات/وكلاء). لا يمكن للمدير حذف
  حسابه نفسه من اللوحة.

## أمان لاحق (موصى به قبل التوزيع العام الواسع)
- شهادة TLS حقيقية (دومين + Let's Encrypt) وتفعيل التحقق في التطبيق.
- التحكم بالتسجيل (تأكيد إيميل / دعوات) بدل التسجيل المفتوح.
- soft-delete (حذف مؤجّل قابل للاسترجاع) + تحديد معدّل للطلبات.
